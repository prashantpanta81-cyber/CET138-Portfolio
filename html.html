HTML

HTML (HyperText Markup Language) is the skeleton of every webpage. HTML5, the current standard, introduced semantic elements that make pages more accessible and meaningful to search engines.

HTML5 Boilerplate & Semantic Elements

Every HTML5 page starts with a standard structure. Semantic elements like <header>, <main>, and <footer> describe the purpose of content — unlike a generic <div>.

<!-- HTML5 boilerplate -->
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Page Title</title>
  <link rel="stylesheet" href="style.css">
</head>
<body>

  <header>               <!-- site header / nav -->
    <nav>...</nav>
  </header>

  <main>               <!-- primary page content -->
    <article>
      <h1>Heading</h1>
      <p>Paragraph text.</p>
    </article>

    <aside>Related content</aside>
  </main>

  <footer>© 2025</footer>

</body>
</html>
ElementPurposeInstead of
<header>Site or section header — usually contains nav<div id="header">
<nav>Navigation links block<div class="nav">
<main>The primary unique content of the page<div id="content">
<article>Self-contained content (blog post, card)<div class="post">
<section>Thematic grouping with a heading<div class="section">
<aside>Tangentially related content / sidebar<div class="sidebar">
<footer>Page or section footer<div id="footer">
Live HTML Form Demo

Forms collect user input. Each <input> has a type attribute that controls its behaviour, and a label linked via for/id for accessibility.

<form action="/submit" method="POST">

  <!-- label's 'for' must match input's 'id' -->
  <label for="name">Full Name</label>
  <input type="text" id="name" name="name" required>

  <label for="email">Email</label>
  <input type="email" id="email" name="email" required>

  <button type="submit">Submit</button>
</form>

Multimedia Elements

HTML5 added native <video>, <audio>, and improved <img> elements — no plugins required. The alt attribute on images is critical for accessibility.

<!-- Image with alt text -->
<img src="photo.jpg"
     alt="A student coding at a desk"
     width="600" height="400"
     loading="lazy">

<!-- HTML5 Video (multiple formats for browser support) -->
<video controls width="640" poster="thumbnail.jpg">
  <source src="video.mp4"  type="video/mp4">
  <source src="video.webm" type="video/webm">
  Your browser does not support the video element.
</video>

<!-- HTML5 Audio -->
<audio controls>
  <source src="audio.mp3" type="audio/mpeg">
  <source src="audio.ogg" type="audio/ogg">
</audio>

<!-- Figure with caption -->
<figure>
  <img src="chart.png" alt="Bar chart showing revenue">
  <figcaption>Q4 2024 Revenue by Region</figcaption>
</figure>
AttributeUsed onPurpose
alt<img>Screen readers & shown if image fails to load
controls<video>, <audio>Displays browser's built-in play/pause UI
loading="lazy"<img>Defers loading until the image is near the viewport
poster<video>Thumbnail shown before the video plays
autoplay muted<video>Auto-plays (muted required by most browsers)
